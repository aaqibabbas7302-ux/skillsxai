Data Analytics Cheatsheet

Reference guide: frameworks, tools, stats, and visualization

1. Data Analytics Fundamentals

Four levels form a decision ladder: what happened → why → what next (likely) → what to do.

Type Question Examples
Descriptive What happened? Dashboards, reports, trends over time, cohort snapshots
Diagnostic Why did it happen? Drill-downs, variance analysis, funnels, segment comparisons
Predictive What is likely to happen? Forecasting, scoring, propensity models, A/B test lift estimates
Prescriptive What should we do? Optimization, pricing rules, resource allocation, scenario planning
  • Good practice: align metrics to decisions; document definitions (single source of truth).
  • Validate with stakeholders before building predictive/prescriptive layers on shaky data.
Analyst workflow (typical)
  1. Clarify the decision, audience, and success criteria (not just “more dashboard”).
  2. Profile data: row counts, key uniqueness, null rates, min/max dates, grain of fact tables.
  3. Transform in a repeatable layer (dbt, SQL views, Power Query) — avoid one-off spreadsheet fixes for production logic.
  4. Validate with reconciliations (source vs. mart totals) and edge cases (refunds, timezone, currency).
  5. Communicate limitations: sampling, missing data, definitional changes, and freshness.

2. SQL Quick Reference

SELECT & filters
SELECT col1, AGG(col2) AS metric FROM schema.table WHERE date_col >= '2024-01-01' AND status IN ('active','trial') GROUP BY 1 HAVING SUM(amount) > 0 ORDER BY metric DESC LIMIT 100;
JOIN patterns
  • INNER — only matching keys in both tables.
  • LEFT — all left rows; nulls if no match (common for fact + dimension).
  • Avoid accidental duplicates: join on stable keys; dedupe dim tables first.
Subqueries & window functions
-- Correlated: per-user last order SELECT u.id, o.total FROM users u JOIN orders o ON o.id = ( SELECT id FROM orders o2 WHERE o2.user_id = u.id ORDER BY o2.placed_at DESC LIMIT 1 ); -- Windows: running sum & prior row SELECT order_date, revenue, SUM(revenue) OVER (ORDER BY order_date) AS running, LAG(revenue) OVER (ORDER BY order_date) AS prev FROM daily_sales; -- Common: row_number to dedupe SELECT * FROM ( SELECT *, ROW_NUMBER() OVER (PARTITION BY id ORDER BY updated_at DESC) AS rn FROM events ) t WHERE rn = 1;
CTEs (readability & reuse)
WITH daily AS ( SELECT user_id, DATE(ts) AS d, COUNT(*) AS ev FROM events GROUP BY 1,2 ), flagged AS ( SELECT * FROM daily WHERE ev >= 5 ) SELECT * FROM flagged;
CASE, NULLs, sets
CASE WHEN amt < 0 THEN 'refund' WHEN amt IS NULL THEN 'unknown' ELSE 'sale' END -- NULL never equals NULL: use IS NULL SELECT * FROM t WHERE col IS NULL; SELECT id FROM a UNION -- distinct SELECT id FROM b; -- UNION ALL keeps duplicates (faster)
Gotcha Mitigation
Duplicate keys after JOIN Join to dimension at correct grain; dedupe with ROW_NUMBER or DISTINCT ON (Postgres).
Implicit type coercion Cast dates and numerics explicitly; compare VARCHAR ids carefully across systems.

Use EXPLAIN to inspect plans; index filter columns in large tables when possible.

3. Excel for Analytics

Topic How / when
VLOOKUP / XLOOKUP =XLOOKUP(key, keys_col, return_col, "—") — prefer XLOOKUP (exact match, no column index error). VLOOKUP needs lookup column first.
Pivot tables Insert → Pivot: rows = dimensions, values = sum/count. Use Show Values As (%, of parent, running total) for quick variance views.
Conditional formatting Data bars, color scales, or rules like cell < 0 for exception reporting; keep palettes consistent with brand.
Charts Line = trends; column/bar = category comparison; avoid 3D; label axes; truncate long category labels.
Function / feature Notes
INDEX / MATCH Two-way lookup: =INDEX(return_range, MATCH(row_key, colA,0), MATCH(col_key, row1,0)). Safer than VLOOKUP for leftward columns.
SUMIFS / COUNTIFS =SUMIFS(sum_range, crit_range1, crit1, crit_range2, crit2) — multiple AND conditions without array entry in all modern Excel.
Excel Tables (Ctrl+T) Structured references auto-expand; easier for pivot source ranges and fewer broken references when rows are added.

Power Query (Get Data) to clean, merge, and refresh repeatable pipelines without copying formulas ad hoc.

4. Power BI Essentials

  • Model: star schema (facts + dimensions), active relationships, Filter direction (single for normal dims; both ways only with care).
  • Calculated column vs measure: column = row context (stored); measure = aggregate with filter context (typically preferred for KPIs).
  • DAX — filter functions: CALCULATE( [Sales], 'Region'[Name]="West" ); ALL, ALLEXCEPT, REMOVEFILTERS to adjust filter context.
  • Time intel: DATESYTD, SAMEPERIODLASTYEAR, DATESBETWEEN (requires a date table marked as date table).
  • Visuals: table/matrix for details; line for trends; bar for ranks; map for geo; avoid pie for many slices — use bar.
  • Performance: reduce high-cardinality columns in visuals; use aggregations; DirectQuery only when data volume/real-time demands it.
DAX patterns (copy-paste starters)
-- Reuse filter context; add slice Sales West := CALCULATE( [Sales], 'Region'[Name] = "West" ) -- Remove filters on Product (compare to total same period) Sales All Products := CALCULATE( [Sales], REMOVEFILTERS( 'Product' ) ) -- YoY same measure (needs Date table relationship) Sales PY := CALCULATE( [Sales], SAMEPERIODLASTYEAR( 'Date'[Date] ) ) -- Running total in visual (by date) Sales YTD := TOTALYTD( [Sales], 'Date'[Date] )

5. Statistical Concepts

Measure Meaning & when it matters
Mean (μ) Average; sensitive to outliers. Pair with distribution plot.
Median 50th percentile; robust for skewed $ data (AOV, salary).
Mode Most frequent; useful for categorical or binned data.
Std dev (σ) Spread around mean. High σ → volatile; use with same units as the variable.
Correlation (r) Linear association (−1 to 1). Not causation; confounders and range restrictions distort r.
Percentiles (p25, p75) Robust spread; box plots show IQR (p75−p25). Watch for duplicate values in small samples.
Variance / CV Variance = average squared deviation. Coefficient of variation = σ/μ — compare volatility across different scales.
Sampling bias Convenience samples over/under-represent groups. Always ask: who is missing from this data?
Experiments & A/B tests (essentials)
  • Randomization balances known and unknown confounders; blocking (e.g. by day of week) can reduce noise when traffic is structured.
  • Type I error (α): false positive (declare winner when no effect). Type II (β): false negative. Power = 1−β; increase sample size, reduce variance, or lift effect size to improve power.
  • Define primary metric up front; watch multiple testing / peeking (optional stopping inflates false positives) — pre-register analysis and use consistent exposure windows.
  • Report confidence interval for the lift, not just p-value; consider practical significance (minimum detectable effect that matters to the business).

6. Data Visualization Best Practices

Chart selection (quick map)
  • Change over time → line or area (stack only when parts sum to a meaningful total).
  • Compare categories → sorted bar; few periods → could use small multiples of lines.
  • Part-to-whole (few segments) → horizontal bar; many parts → table + conditional formatting often clearer than pie.
  • Relationship → scatter (size/color for 3rd/4th variable); heatmap for matrix of categories.
Storytelling with data
  1. Lead with the so what — the decision or action implied.
  2. One main message per view; de-clutter; align titles to the insight.
  3. Annotate context (events, baselines) and state data freshness / caveats.
Accessibility & clarity
  • Do not rely on color alone — add labels, patterns, or direct values on marks.
  • Check contrast; colorblind-safe palettes (e.g. blue–orange) for key comparisons.
  • Sort categories by metric (not alphabet) when the story is rank or magnitude.
  • Start bar axis at 0 for length encodings; truncate only with clear break symbol and explanation.

7. Key Metrics & KPIs by Industry

Industry Sample KPIs (define numerator/denominator & time window)
Marketing CAC, LTV, LTV:CAC, ROAS / MER, conversion rate, email CTR, branded vs non-branded search share, pipeline influenced.
Finance Gross margin %, OPEX ratio, DSO / DPO, current ratio, cash conversion cycle, budget variance, revenue recognition alignment.
Operations Throughput, cycle time, FPY / defect rate, OEE, inventory turns, on-time in-full (OTIF), capacity utilization, SLA compliance.
SaaS / Product MRR/ARR, net revenue retention (NRR), logo churn, expansion rate, PQL → paid conversion, activation rate, time-to-value, NPS/CSAT.
Retail / E‑com Same-store sales growth, basket size, sell-through, gross margin return on investment (GMROI), return rate, stockout %, conversion by channel.

Naming: always document whether metrics are trailing 12 months, fiscal vs calendar, gross vs net of discounts, and attribution window for marketing.

Before you ship (quick checklist)

  • Metric definition written in plain language + SQL/DAX snippet or formula.
  • Time zone and business calendar (week start, holidays) explicit.
  • Nulls and refunds handled consistently with finance/ops.
  • Spot-check: hand-calc or small export matches dashboard for a slice.
  • Primary visual answers the title; remove chart junk (gridlines, 3D).
  • Version or “as of” date on shared reports.
  • PII minimized; access follows least privilege.
  • Known limitations listed (lags, estimates, incomplete sources).
